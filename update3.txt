CONCLUSION: Android ≈ 100% parity with web frontend.
Android has MORE features than web due to:
- 20+ confirmation dialogs (web has none)
- Dedicated screens (BackupRestore, NetworkDetail, ManualMode as full screen)
- Better UX (manual connect form instead of prompt(), WPA status display, rotation)
- Bottom navigation with Settings tab (web has top mini-nav)
- All confirmation dialogs for destructive actions (web executes immediately)

Web pages that DON'T EXIST (but are routed):
- /actions.html   → not in web/ directory (Android has actions in NetworkScreen)
- /manual.html    → not in web/ directory (Android has ManualModeScreen)

These two pages are referenced in webapp.py routes but the HTML files are missing.
Android covers both features (actions via NetworkScreen Quick Actions,
manual mode via ManualModeScreen).
